## 一、Go 的 goroutine

### 面试问题

什么是 goroutine？它和线程有什么区别？如何创建和管理 goroutine？

### 标准答案

**基本用法：**

```go
func hello(name string) {
    fmt.Printf("Hello, %s!\n", name)
}

func main() {
    go hello("Alice")  // 启动 goroutine
    go hello("Bob")

    time.Sleep(time.Second)  // 等待 goroutine 完成（不推荐）
}

// 推荐使用 WaitGroup
var wg sync.WaitGroup

func main() {
    wg.Add(2)
    go func() {
        defer wg.Done()
        hello("Alice")
    }()
    go func() {
        defer wg.Done()
        hello("Bob")
    }()
    wg.Wait()  // 等待所有 goroutine 完成
}
```

**goroutine vs 线程：**

| 特性 | goroutine | 线程 |
|------|-----------|------|
| 内存占用 | 约 2-8KB | 约 1-8MB |
| 创建开销 | 极小 | 较大 |
| 调度方式 | 用户态（Go 调度器） | 内核态（OS 调度器） |
| 切换开销 | 约 100ns | 约 1-10µs |
| 数量 | 可以创建数十万个 | 通常几百到几千 |
| 栈大小 | 动态增长（初始 2KB） | 固定（通常 1-8MB） |

**Go 调度器（GMP 模型）：**

```go
// GMP 模型：
// G = Goroutine（协程）
// M = Machine（操作系统线程）
// P = Processor（逻辑处理器）

// P 的数量由 GOMAXPROCS 决定（默认等于 CPU 核数）
runtime.GOMAXPROCS(4)  // 使用 4 个逻辑处理器

// 调度流程：
// 1. 新建 G，放入 P 的本地队列
// 2. M 从 P 的本地队列获取 G 执行
// 3. G 阻塞时，M 与 P 解绑，P 绑定新的 M 继续执行其他 G
// 4. 本地队列空时，从全局队列或其他 P 偷取 G（work-stealing）

// GOMAXPROCS 的影响
runtime.GOMAXPROCS(1)  // 单核，goroutine 并发但不并行
runtime.GOMAXPROCS(4)  // 多核，goroutine 可以并行执行
```

**goroutine 的注意事项：**

```go
// 1. 主 goroutine 退出，所有 goroutine 都会退出
func main() {
    go func() {
        time.Sleep(time.Second)
        fmt.Println("hello")  // 不会执行
    }()
    // main 退出，goroutine 被杀
}

// 2. goroutine 泄漏
func leak() {
    ch := make(chan int)
    go func() {
        ch <- 1  // 没有接收者，永远阻塞
    }()
    // ch 没有被关闭，goroutine 泄漏
}

// 3. 控制 goroutine 数量
sem := make(chan struct{}, 10)  // 最多 10 个并发

for i := 0; i < 1000; i++ {
    sem <- struct{}{}  // 获取令牌
    go func(i int) {
        defer func() { <-sem }()  // 释放令牌
        doWork(i)
    }(i)
}

// 或使用 errgroup
g, ctx := errgroup.WithContext(context.Background())
g.SetLimit(10)  // 最多 10 个并发

for i := 0; i < 1000; i++ {
    i := i
    g.Go(func() error {
        return doWork(ctx, i)
    })
}

if err := g.Wait(); err != nil {
    log.Fatal(err)
}
```

## 二、Go 的 channel

### 面试问题

channel 是什么？有缓冲和无缓冲有什么区别？如何使用 channel 实现同步？

### 标准答案

**基本用法：**

```go
// 创建 channel
ch := make(chan int)      // 无缓冲
ch := make(chan int, 10)  // 带缓冲，容量 10

// 发送
ch <- 42

// 接收
v := <-ch
v, ok := <-ch  // ok 为 false 表示 channel 已关闭

// 关闭
close(ch)

// 遍历
for v := range ch {
    fmt.Println(v)
}
// 当 channel 关闭且缓冲区为空时，循环结束
```

**无缓冲 vs 有缓冲：**

```go
// 无缓冲：发送和接收必须同时就绪（同步）
ch := make(chan int)
go func() {
    ch <- 42  // 阻塞，直到有人接收
}()
v := <-ch  // 阻塞，直到有人发送

// 有缓冲：缓冲区满时发送阻塞，空时接收阻塞（异步）
ch := make(chan int, 2)
ch <- 1  // 不阻塞
ch <- 2  // 不阻塞
ch <- 3  // 阻塞，缓冲区满

v := <-ch  // 不阻塞
v := <-ch  // 不阻塞
v := <-ch  // 阻塞，缓冲区空
```

**channel 的方向：**

```go
// 双向 channel
ch := make(chan int)

// 只写 channel
func producer(ch chan<- int) {
    ch <- 42
}

// 只读 channel
func consumer(ch <-chan int) {
    v := <-ch
    fmt.Println(v)
}

func main() {
    ch := make(chan int)
    go producer(ch)
    consumer(ch)
}
```

**常见的 channel 模式：**

```go
// 1. 生产者-消费者
func producer(ch chan<- int) {
    for i := 0; i < 10; i++ {
        ch <- i
    }
    close(ch)
}

func consumer(ch <-chan int) {
    for v := range ch {
        fmt.Println(v)
    }
}

func main() {
    ch := make(chan int)
    go producer(ch)
    consumer(ch)
}

// 2. 超时控制
ch := make(chan int, 1)
go func() {
    time.Sleep(2 * time.Second)
    ch <- 42
}()

select {
case v := <-ch:
    fmt.Println("received:", v)
case <-time.After(1 * time.Second):
    fmt.Println("timeout")
}

// 3. 非阻塞发送
ch := make(chan int, 1)
select {
case ch <- 42:
    fmt.Println("sent")
default:
    fmt.Println("channel full")
}

// 4. 非阻塞接收
ch := make(chan int, 1)
select {
case v := <-ch:
    fmt.Println("received:", v)
default:
    fmt.Println("no data")
}

// 5. done channel
done := make(chan struct{})
go func() {
    defer close(done)
    // 做一些工作
}()
<-done  // 等待完成

// 6. fan-out fan-in
func fanOut(input <-chan int, workers int) []<-chan int {
    channels := make([]<-chan int, workers)
    for i := 0; i < workers; i++ {
        channels[i] = worker(input)
    }
    return channels
}

func fanIn(channels ...<-chan int) <-chan int {
    var wg sync.WaitGroup
    merged := make(chan int)
    for _, ch := range channels {
        wg.Add(1)
        go func(ch <-chan int) {
            defer wg.Done()
            for v := range ch {
                merged <- v
            }
        }(ch)
    }
    go func() {
        wg.Wait()
        close(merged)
    }()
    return merged
}
```

## 三、Go 的 select

### 面试问题

select 是什么？和 switch 有什么区别？

### 标准答案

**基本用法：**

```go
// select 用于多路复用 channel
ch1 := make(chan string)
ch2 := make(chan string)

go func() {
    time.Sleep(1 * time.Second)
    ch1 <- "one"
}()

go func() {
    time.Sleep(2 * time.Second)
    ch2 <- "two"
}()

select {
case msg := <-ch1:
    fmt.Println("received:", msg)
case msg := <-ch2:
    fmt.Println("received:", msg)
case <-time.After(3 * time.Second):
    fmt.Println("timeout")
}
```

**select vs switch：**

| 特性 | select | switch |
|------|--------|--------|
| 用途 | channel 多路复用 | 值匹配 |
| 阻塞 | 可能阻塞 | 不阻塞 |
| 随机选择 | 有多个就绪时随机选一个 | 按顺序匹配 |
| default | 非阻塞 | 其他分支 |

**select 的规则：**

```go
// 1. 没有 default 时，所有 case 都不就绪则阻塞
select {
case v := <-ch:
    fmt.Println(v)
// 没有 default，如果 ch 没有数据则阻塞
}

// 2. 有 default 时，所有 case 都不就绪则执行 default（非阻塞）
select {
case v := <-ch:
    fmt.Println(v)
default:
    fmt.Println("no data")
}

// 3. 多个 case 就绪时，随机选择一个
select {
case v := <-ch1:
    fmt.Println("ch1:", v)
case v := <-ch2:
    fmt.Println("ch2:", v)
}

// 4. 空 select 永远阻塞
select {}  // 程序卡死

// 5. 超时控制
select {
case v := <-ch:
    fmt.Println(v)
case <-time.After(5 * time.Second):
    fmt.Println("timeout")
}

// 6. 取消操作
ctx, cancel := context.WithCancel(context.Background())
defer cancel()

select {
case v := <-ch:
    fmt.Println(v)
case <-ctx.Done():
    fmt.Println("cancelled")
}
```

**select 的常见用法：**

```go
// 1. 定时器
ticker := time.NewTicker(time.Second)
defer ticker.Stop()

for {
    select {
    case <-ticker.C:
        fmt.Println("tick")
    case <-quit:
        return
    }
}

// 2. 心跳
heartbeat := time.NewTicker(500 * time.Millisecond)
defer heartbeat.Stop()

for {
    select {
    case <-heartbeat.C:
        // 发送心跳
        conn.Write([]byte("ping"))
    case data := <-dataCh:
        // 处理数据
        process(data)
    }
}

// 3. 优雅退出
quit := make(chan os.Signal, 1)
signal.Notify(quit, syscall.SIGINT, syscall.SIGTERM)

select {
case <-quit:
    fmt.Println("shutting down...")
    // 清理资源
}
```

## 四、Go 的 sync 包

### 面试问题

Go 的 sync 包有哪些常用类型？如何实现并发安全？

### 标准答案

**Mutex（互斥锁）：**

```go
type SafeCounter struct {
    mu    sync.Mutex
    count int
}

func (c *SafeCounter) Increment() {
    c.mu.Lock()
    defer c.mu.Unlock()
    c.count++
}

func (c *SafeCounter) Value() int {
    c.mu.Lock()
    defer c.mu.Unlock()
    return c.count
}
```

**RWMutex（读写锁）：**

```go
type SafeMap struct {
    mu sync.RWMutex
    m  map[string]int
}

func (s *SafeMap) Get(key string) int {
    s.mu.RLock()
    defer s.mu.RUnlock()
    return s.m[key]
}

func (s *SafeMap) Set(key string, value int) {
    s.mu.Lock()
    defer s.mu.Unlock()
    s.m[key] = value
}

// 读写锁适用于读多写少的场景
// 读锁之间不互斥，读写互斥，写写互斥
```

**WaitGroup（等待组）：**

```go
var wg sync.WaitGroup

for i := 0; i < 5; i++ {
    wg.Add(1)
    go func(i int) {
        defer wg.Done()
        fmt.Println(i)
    }(i)
}

wg.Wait()  // 等待所有 goroutine 完成
```

**Once（单次执行）：**

```go
var once sync.Once
var instance *Singleton

func GetInstance() *Singleton {
    once.Do(func() {
        instance = &Singleton{}
    })
    return instance
}

// 确保初始化只执行一次，即使多个 goroutine 同时调用
```

**Pool（对象池）：**

```go
var pool = sync.Pool{
    New: func() interface{} {
        return new(bytes.Buffer)
    },
}

func process(data []byte) {
    buf := pool.Get().(*bytes.Buffer)
    defer pool.Put(buf)

    buf.Reset()
    buf.Write(data)
    // 使用 buf...
}

// 适用场景：
// 1. 高频创建和销毁的对象
// 2. 临时对象（如 bytes.Buffer）
// 不适用：
// 1. 需要长期持有的对象
// 2. 池中对象状态不一致
```

**Cond（条件变量）：**

```go
var mu sync.Mutex
var cond = sync.NewCond(&mu)
var ready bool

func waitForReady() {
    mu.Lock()
    for !ready {
        cond.Wait()  // 释放锁并等待
    }
    mu.Unlock()
}

func setReady() {
    mu.Lock()
    ready = true
    cond.Signal()  // 唤醒一个等待的 goroutine
    // cond.Broadcast()  // 唤醒所有等待的 goroutine
    mu.Unlock()
}
```

**sync.Map：**

```go
var m sync.Map

// 存储
m.Store("key", "value")

// 加载
v, ok := m.Load("key")

// 加载或存储
v, loaded := m.LoadOrStore("key", "default")

// 删除
m.Delete("key")

// 遍历
m.Range(func(k, v interface{}) bool {
    fmt.Println(k, v)
    return true  // 返回 false 停止遍历
})

// 适用场景：
// 1. key 集合稳定（读多写少）
// 2. 不同 goroutine 操作不同的 key
```

**sync.Once vs init：**

```go
// init：程序启动时执行，不可延迟
func init() {
    // 初始化
}

// sync.Once：首次调用时执行，可延迟
var once sync.Once
func getInstance() *DB {
    once.Do(func() {
        db = connectDB()
    })
    return db
}
```

## 五、Go 的 context

### 面试问题

context 是什么？有哪些类型？如何使用 context 控制 goroutine 的生命周期？

### 标准答案

**基本用法：**

```go
// context 用于在 goroutine 之间传递取消信号、超时、截止时间

// 创建 context
ctx := context.Background()  // 根 context，永远不会取消
ctx := context.TODO()        // 不确定用什么 context 时使用

// 派生 context
ctx, cancel := context.WithCancel(parentCtx)
ctx, cancel := context.WithTimeout(parentCtx, 5*time.Second)
ctx, cancel := context.WithDeadline(parentCtx, time.Now().Add(5*time.Second))
ctx = context.WithValue(parentCtx, key, value)
```

**WithCancel：**

```go
ctx, cancel := context.WithCancel(context.Background())
defer cancel()  // 确保取消

go func() {
    for {
        select {
        case <-ctx.Done():
            fmt.Println("cancelled:", ctx.Err())
            return
        default:
            // 做一些工作
            time.Sleep(100 * time.Millisecond)
        }
    }
}()

time.Sleep(time.Second)
cancel()  // 取消 goroutine
```

**WithTimeout：**

```go
ctx, cancel := context.WithTimeout(context.Background(), 2*time.Second)
defer cancel()

select {
case result := <-doSomething(ctx):
    fmt.Println("result:", result)
case <-ctx.Done():
    fmt.Println("timeout:", ctx.Err())
}

func doSomething(ctx context.Context) <-chan string {
    ch := make(chan string, 1)
    go func() {
        time.Sleep(3 * time.Second)  // 超过超时时间
        ch <- "done"
    }()
    return ch
}
```

**WithValue：**

```go
type contextKey string

const userIDKey contextKey = "userID"

func middleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        userID := r.Header.Get("X-User-ID")
        ctx := context.WithValue(r.Context(), userIDKey, userID)
        next.ServeHTTP(w, r.WithContext(ctx))
    })
}

func handler(w http.ResponseWriter, r *http.Request) {
    userID := r.Context().Value(userIDKey).(string)
    fmt.Fprintf(w, "Hello, %s", userID)
}
```

**context 的最佳实践：**

```go
// 1. context 作为函数的第一个参数
func DoSomething(ctx context.Context, arg Arg) error {
    // ...
}

// 2. 不要将 context 存储在结构体中
type Bad struct {
    ctx context.Context  // 不要这样做
}

// 3. 不要传递 nil context
func DoSomething(ctx context.Context) error {
    // 如果不确定用什么，用 context.TODO()
}

// 4. context 只用于传递请求范围的数据
// 不要用于传递函数的可选参数

// 5. context.Value 应该很少使用
// 用于传递请求级的元数据（如 trace ID、user ID）
// 不要用于传递业务数据
```

**面试加分点：** context 是 Go 并发编程的核心。当一个 goroutine 创建了子 goroutine 时，父 goroutine 可以通过 context 通知所有子 goroutine 取消。这在 HTTP 请求处理、数据库查询等场景中非常重要。
