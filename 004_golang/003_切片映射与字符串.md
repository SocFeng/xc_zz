## 一、Go 的数组和切片

### 面试问题

Go 的数组和切片有什么区别？切片的底层原理是什么？有哪些注意事项？

### 标准答案

**数组：**

```go
// 数组是值类型，长度固定
var arr [5]int           // [0, 0, 0, 0, 0]
arr := [5]int{1, 2, 3, 4, 5}
arr := [...]int{1, 2, 3} // 编译器自动计算长度

// 数组是值类型，赋值会拷贝
a := [3]int{1, 2, 3}
b := a          // b 是 a 的拷贝
b[0] = 100
fmt.Println(a)  // [1, 2, 3]  不受影响

// 数组比较（长度和元素都相同才能比较）
a := [3]int{1, 2, 3}
b := [3]int{1, 2, 3}
fmt.Println(a == b)  // true
```

**切片：**

```go
// 切片是引用类型，动态长度
var s []int              // nil 切片
s := []int{1, 2, 3}     // 字面量
s := make([]int, 5)     // len=5, cap=5
s := make([]int, 5, 10) // len=5, cap=10

// 切片是数组的视图
arr := [5]int{1, 2, 3, 4, 5}
s := arr[1:3]  // [2, 3]，指向 arr 的第 1-2 个元素
s[0] = 20
fmt.Println(arr)  // [1, 20, 3, 4, 5]  原数组被修改
```

**切片的底层结构：**

```go
// 切片的底层结构（runtime/slice.go）
type slice struct {
    array unsafe.Pointer  // 指向底层数组的指针
    len   int             // 长度
    cap   int             // 容量
}

// 切片操作
s := make([]int, 3, 5)
// s.array → [0, 0, 0, ?, ?]
// s.len = 3
// s.cap = 5
```

**切片的扩容机制：**

```go
// append 超过容量时会扩容
s := make([]int, 0, 2)
s = append(s, 1)  // len=1, cap=2
s = append(s, 2)  // len=2, cap=2
s = append(s, 3)  // len=3, cap=4  扩容！

// 扩容策略（Go 1.18+）：
// - 新容量 < 256：翻倍
// - 新容量 >= 256：增长 (newCap + 3*256) / 4
// 扩容后会分配新的底层数组，原数组不变

// 扩容会导致切片与原数组断开关系
arr := [3]int{1, 2, 3}
s := arr[:]
s = append(s, 4)  // 扩容，s 指向新数组
s[0] = 100
fmt.Println(arr)  // [1, 2, 3]  不受影响
```

**切片的注意事项：**

```go
// 1. 切片是引用类型
a := []int{1, 2, 3}
b := a
b[0] = 100
fmt.Println(a)  // [100, 2, 3]  被修改

// 2. 子切片共享底层数组
a := []int{1, 2, 3, 4, 5}
b := a[1:3]  // [2, 3]
b[0] = 20
fmt.Println(a)  // [1, 20, 3, 4, 5]  被修改

// 3. copy 是深拷贝
a := []int{1, 2, 3}
b := make([]int, len(a))
copy(b, a)
b[0] = 100
fmt.Println(a)  // [1, 2, 3]  不受影响

// 4. 切片判空
var s []int
fmt.Println(s == nil)  // true
fmt.Println(len(s))    // 0

s = []int{}
fmt.Println(s == nil)  // false
fmt.Println(len(s))    // 0

// 推荐用 len 判空
if len(s) == 0 {
    // 空切片
}

// 5. 切片删除元素
s := []int{1, 2, 3, 4, 5}
// 删除索引 2 的元素
s = append(s[:2], s[3:]...)  // [1, 2, 4, 5]

// 6. 切片的陷阱：内存泄漏
func getNames(users []User) []string {
    names := make([]string, len(users))
    for i, u := range users {
        names[i] = u.Name
    }
    return names
    // 如果 users 很大，names 只取了 Name
    // 但 names 底层可能引用了 users 的底层数组
    // 解决：使用 append 而不是预分配
}

func getNames(users []User) []string {
    names := make([]string, 0, len(users))
    for _, u := range users {
        names = append(names, u.Name)
    }
    return names
}
```

## 二、Go 的 map

### 面试问题

Go 的 map 底层原理是什么？有哪些注意事项？map 是线程安全的吗？

### 标准答案

**基本用法：**

```go
// 创建
m := make(map[string]int)
m := map[string]int{"a": 1, "b": 2}

// 增改
m["c"] = 3

// 查
v := m["a"]       // 1
v, ok := m["x"]   // 0, false（不存在返回零值）

// 删
delete(m, "a")

// 遍历（无序）
for k, v := range m {
    fmt.Println(k, v)
}

// 长度
len(m)
```

**map 的底层原理：**

```go
// map 底层是哈希表（runtime/map.go）
// 结构：hmap
type hmap struct {
    count     int     // 元素个数
    B         uint8   // 2^B 个桶
    buckets   unsafe.Pointer  // 桶数组
    oldbuckets unsafe.Pointer // 扩容时的旧桶
    // ...
}

// 每个桶（bmap）最多存 8 个键值对
// 超过 8 个会使用溢出桶

// 查找过程：
// 1. 计算 key 的哈希值
// 2. 用哈希值的低 B 位定位桶
// 3. 用哈希值的高 8 位快速比较
// 4. 在桶中顺序查找 key
```

**map 的注意事项：**

```go
// 1. map 不是线程安全的
m := make(map[string]int)
// 并发读写会 panic: concurrent map read and map write

// 解决方案：
// 方案一：sync.Mutex
var mu sync.Mutex
m := make(map[string]int)

func set(key string, value int) {
    mu.Lock()
    defer mu.Unlock()
    m[key] = value
}

func get(key string) int {
    mu.Lock()
    defer mu.Unlock()
    return m[key]
}

// 方案二：sync.RWMutex（读多写少场景）
var rwmu sync.RWMutex

func get(key string) int {
    rwmu.RLock()
    defer rwmu.RUnlock()
    return m[key]
}

// 方案三：sync.Map（官方并发安全 map）
var sm sync.Map
sm.Store("key", "value")
v, ok := sm.Load("key")
sm.Delete("key")
sm.Range(func(k, v interface{}) bool {
    fmt.Println(k, v)
    return true  // 返回 false 停止遍历
})

// 2. map 的 key 必须是可比较的类型
// 可以：int, string, bool, 指针, 数组, struct（字段都可比较）
// 不可以：slice, map, func

// 3. map 的遍历顺序是随机的
m := map[string]int{"a": 1, "b": 2, "c": 3}
for range m {
    // 每次运行顺序可能不同
}
// Go 故意这样做，防止依赖遍历顺序

// 4. nil map 可以读但不能写
var m map[string]int
v := m["key"]  // OK，返回零值
m["key"] = 1   // panic: assignment to entry in nil map

// 5. map 不支持取地址
m := map[string]int{"a": 1}
// &m["a"]  // 编译错误

// 6. 嵌套 map 需要手动初始化
m := make(map[string]map[string]int)
// m["a"]["b"] = 1  // panic: nil map
m["a"] = make(map[string]int)
m["a"]["b"] = 1  // OK
```

**sync.Map 的适用场景：**

```go
// 适合：
// 1. key 集合基本稳定（读多写少）
// 2. 不同 goroutine 操作不同的 key

// 不适合：
// 1. key 频繁增删
// 2. 需要遍历（性能较差）

// 相比 Mutex+Map，sync.Map 的优势：
// - 读操作无锁（Load）
// - 不需要初始化
// - 内置 dirty 优化
```

## 三、Go 的字符串

### 面试问题

Go 的字符串底层是什么？如何处理中文？string 和 []byte 有什么区别？

### 标准答案

**字符串的底层：**

```go
// 字符串底层结构（runtime/string.go）
type stringHeader struct {
    Data unsafe.Pointer  // 指向字节数组的指针
    Len  int             // 字节长度
}

// 字符串是不可变的字节序列
s := "hello"
// s[0] = 'H'  // 编译错误：cannot assign to s[0]

// 字符串长度
s := "hello"
len(s)  // 5（字节数）

s := "你好"
len(s)  // 6（字节数，每个中文 3 字节）
utf8.RuneCountInString(s)  // 2（字符数）
```

**string vs []byte：**

```go
// string 是不可变的，[]byte 是可变的
s := "hello"
b := []byte(s)
b[0] = 'H'
s2 := string(b)  // "Hello"

// 转换会拷贝数据
s := "hello"
b := []byte(s)  // 拷贝
b[0] = 'H'
fmt.Println(s)  // "hello"  不受影响

// 高效转换（零拷贝，但要注意安全）
// unsafe 方式（不推荐，除非性能关键）
func stringToBytes(s string) []byte {
    return *(*[]byte)(unsafe.Pointer(&s))
}

func bytesToString(b []byte) string {
    return *(*string)(unsafe.Pointer(&b))
}
```

**处理中文（Unicode）：**

```go
s := "你好，世界"

// 按字节遍历
for i := 0; i < len(s); i++ {
    fmt.Printf("%x ", s[i])  // e4 bd a0 e5 a5 bd ...
}

// 按 rune（字符）遍历
for i, r := range s {
    fmt.Printf("索引%d: %c\n", i, r)  // 你 好 ， 世 界
}

// rune 切片
runes := []rune(s)
fmt.Println(len(runes))  // 5
runes[0] = '她'
fmt.Println(string(runes))  // 她好，世界

// 字符串操作
import "strings"

strings.Contains(s, "你好")    // true
strings.HasPrefix(s, "你好")   // true
strings.HasSuffix(s, "世界")   // true
strings.Index(s, "，")        // 6
strings.Replace(s, "世界", "Go", -1)
strings.Split("a,b,c", ",")   // ["a", "b", "c"]
strings.Join([]string{"a", "b"}, ",")  // "a,b"
strings.ToUpper("hello")      // "HELLO"
strings.TrimSpace("  hello  ") // "hello"
```

**字符串拼接：**

```go
// 1. + 号（少量拼接）
s := "hello" + " " + "world"

// 2. fmt.Sprintf（格式化）
s := fmt.Sprintf("%s %s", "hello", "world")

// 3. strings.Builder（推荐，大量拼接）
var builder strings.Builder
for i := 0; i < 1000; i++ {
    builder.WriteString("hello")
}
s := builder.String()

// 4. strings.Join（已知切片）
s := strings.Join([]string{"hello", "world"}, " ")

// 5. bytes.Buffer
var buf bytes.Buffer
for i := 0; i < 1000; i++ {
    buf.WriteString("hello")
}
s := buf.String()

// 性能对比：strings.Builder > bytes.Buffer > + > fmt.Sprintf
```

**面试加分点：** Go 的字符串是 UTF-8 编码的。`len()` 返回字节数，`utf8.RuneCountInString()` 返回字符数。遍历字符串时 `range` 按 rune 遍历，索引按字节。
