# Go 流程控制与函数

---

## 一、Go 的流程控制

### 面试问题

Go 的 if、for、switch 和其他语言有什么区别？

### 标准答案

**if 语句：**

```go
// Go 的 if 可以有初始化语句
if err := doSomething(); err != nil {
    fmt.Println("error:", err)
}

// 注意：err 的作用域仅在 if 块内
// fmt.Println(err)  // 编译错误

// 多条件
if x > 0 {
    // ...
} else if x == 0 {
    // ...
} else {
    // ...
}
```

**for 循环（Go 只有 for，没有 while）：**

```go
// 传统 for
for i := 0; i < 10; i++ {
    fmt.Println(i)
}

// while 风格
for i < 10 {
    fmt.Println(i)
    i++
}

// 无限循环
for {
    // break 退出
    if done {
        break
    }
}

// range 遍历
for i, v := range slice {
    fmt.Println(i, v)
}

for k, v := range m {
    fmt.Println(k, v)
}

for i, c := range "hello" {
    fmt.Println(i, c)  // i 是字节索引，c 是 rune
}

// 只要索引或值
for i := range slice { ... }
for _, v := range slice { ... }

// 遍历 channel
for v := range ch {
    fmt.Println(v)
}
```

**range 的坑：**

```go
// 坑1：range 变量是值拷贝
type User struct {
    Name string
}
users := []User{{"Alice"}, {"Bob"}}
for _, u := range users {
    u.Name = "Modified"  // 不会修改原 slice
}

// 正确做法：使用索引
for i := range users {
    users[i].Name = "Modified"
}

// 坑2：range 遍历 map 是无序的
m := map[string]int{"a": 1, "b": 2, "c": 3}
for k, v := range m {
    // 每次运行顺序可能不同
}

// 坑3：range 遍历 slice 是值拷贝
items := make([]*int, 3)
for i, v := range []int{1, 2, 3} {
    items[i] = &v  // 所有 items[i] 都指向同一个 v
}
// 正确做法
for i := 0; i < 3; i++ {
    v := i + 1
    items[i] = &v
}
```

**switch 语句：**

```go
// 基本 switch（不需要 break，自动 break）
switch day {
case "Monday":
    fmt.Println("周一")
case "Tuesday", "Wednesday":  // 多个值
    fmt.Println("工作日")
default:
    fmt.Println("其他")
}

// 带初始化语句
switch os := runtime.GOOS; os {
case "darwin":
    fmt.Println("macOS")
case "linux":
    fmt.Println("Linux")
}

// 无条件 switch（替代 if-else 链）
switch {
case score >= 90:
    grade = "A"
case score >= 80:
    grade = "B"
case score >= 70:
    grade = "C"
default:
    grade = "D"
}

// type switch（类型断言）
func describe(i interface{}) {
    switch v := i.(type) {
    case int:
        fmt.Printf("int: %d\n", v)
    case string:
        fmt.Printf("string: %s\n", v)
    case bool:
        fmt.Printf("bool: %t\n", v)
    default:
        fmt.Printf("unknown: %T\n", v)
    }
}

// fallthrough（穿透到下一个 case）
switch n {
case 1:
    fmt.Println("1")
    fallthrough
case 2:
    fmt.Println("2")  // n=1 时也会执行
}
```

**break 和 continue 标签：**

```go
// 带标签的 break 和 continue
outer:
for i := 0; i < 10; i++ {
    for j := 0; j < 10; j++ {
        if j == 5 {
            break outer  // 跳出外层循环
        }
    }
}

outer:
for i := 0; i < 10; i++ {
    for j := 0; j < 10; j++ {
        if j == 5 {
            continue outer  // 继续外层循环
        }
    }
}
```

## 二、Go 的函数

### 面试问题

Go 的函数有哪些特性？多返回值、匿名函数、闭包是什么？

### 标准答案

**基本函数：**

```go
func add(a, b int) int {
    return a + b
}

// 多返回值
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("division by zero")
    }
    return a / b, nil
}

result, err := divide(10, 3)
if err != nil {
    log.Fatal(err)
}
```

**命名返回值：**

```go
// 命名返回值（naked return）
func divide(a, b float64) (result float64, err error) {
    if b == 0 {
        err = errors.New("division by zero")
        return  // naked return，自动返回 result 和 err
    }
    result = a / b
    return  // naked return
}
```

**可变参数：**

```go
func sum(nums ...int) int {
    total := 0
    for _, n := range nums {
        total += n
    }
    return total
}

fmt.Println(sum(1, 2, 3, 4, 5))  // 15

// 展开 slice
nums := []int{1, 2, 3}
fmt.Println(sum(nums...))  // 展开为 sum(1, 2, 3)
```

**匿名函数：**

```go
// 匿名函数
add := func(a, b int) int {
    return a + b
}
fmt.Println(add(1, 2))

// 立即执行
func() {
    fmt.Println("hello")
}()

// 函数类型
type MathFunc func(int, int) int

func apply(a, b int, f MathFunc) int {
    return f(a, b)
}

fmt.Println(apply(1, 2, add))
```

**闭包：**

```go
func counter() func() int {
    count := 0
    return func() int {
        count++
        return count
    }
}

c := counter()
fmt.Println(c()) // 1
fmt.Println(c()) // 2
fmt.Println(c()) // 3

// 闭包的坑：循环中的闭包
funcs := make([]func(), 5)
for i := 0; i < 5; i++ {
    funcs[i] = func() {
        fmt.Println(i)  // 所有闭包都引用同一个 i
    }
}
for _, f := range funcs {
    f()  // 全部输出 5
}

// 正确做法：使用参数捕获
for i := 0; i < 5; i++ {
    funcs[i] = func(n int) func() {
        return func() {
            fmt.Println(n)
        }
    }(i)
}
// 或者在循环内创建新变量
for i := 0; i < 5; i++ {
    i := i  // 创建新变量
    funcs[i] = func() {
        fmt.Println(i)
    }
}
```

## 三、Go 的 defer

### 面试问题

defer 的执行顺序是什么？有哪些注意事项和常见用法？

### 标准答案

**基本特性：**

```go
// defer 在函数返回前执行（LIFO - 后进先出）
func main() {
    defer fmt.Println("1")
    defer fmt.Println("2")
    defer fmt.Println("3")
    fmt.Println("main")
}
// 输出：main 3 2 1
```

**defer 的参数在声明时求值：**

```go
func main() {
    x := 10
    defer fmt.Println(x)  // x 在此时被求值为 10
    x = 20
    fmt.Println(x)  // 20
}
// 输出：20 10
```

**defer 与闭包：**

```go
func main() {
    x := 10
    defer func() {
        fmt.Println(x)  // 闭包引用 x，此时 x=20
    }()
    x = 20
}
// 输出：20
```

**defer 的执行时机：**

```go
// defer 在 return 语句之后、函数返回之前执行
func f() (result int) {
    defer func() {
        result++
    }()
    return 0  // 实际返回 1
}

// 等价于
func f() (result int) {
    result = 0    // 1. 赋值
    result++      // 2. defer 修改
    return result // 3. 返回
}
```

**常见用法：**

```go
// 1. 资源释放
func readFile(filename string) error {
    f, err := os.Open(filename)
    if err != nil {
        return err
    }
    defer f.Close()  // 确保文件关闭

    // 读取文件...
    return nil
}

// 2. 解锁
func process() {
    mu.Lock()
    defer mu.Unlock()

    // 处理数据...
}

// 3. 计时
func timeTrack(start time.Time, name string) {
    elapsed := time.Since(start)
    fmt.Printf("%s took %s\n", name, elapsed)
}

func slowFunction() {
    defer timeTrack(time.Now(), "slowFunction")
    // ...
}

// 4. 恢复 panic
func safeFunction() {
    defer func() {
        if r := recover(); r != nil {
            fmt.Println("recovered:", r)
        }
    }()
    panic("something went wrong")
}

// 5. 修改命名返回值
func divide(a, b int) (result int, err error) {
    defer func() {
        if r := recover(); r != nil {
            err = fmt.Errorf("panic: %v", r)
        }
    }()
    return a / b, nil
}
```

**defer 的性能注意：**

```go
// 在循环中使用 defer 会有性能开销
// 不推荐
for _, file := range files {
    f, _ := os.Open(file)
    defer f.Close()  // 所有 defer 在函数结束时才执行
}

// 推荐：封装成函数
for _, file := range files {
    func() {
        f, _ := os.Open(file)
        defer f.Close()
        // 处理文件...
    }()
}
```

## 四、Go 的 panic 和 recover

### 面试问题

panic 和 recover 是什么？和 error 有什么区别？什么时候使用？

### 标准答案

**panic（恐慌）：**

```go
// panic 会导致程序崩溃
func divide(a, b int) int {
    if b == 0 {
        panic("division by zero")
    }
    return a / b
}

// 触发 panic 的场景
// 1. 显式调用 panic()
// 2. 数组越界
// 3. 空指针解引用
// 4. 向已关闭的 channel 发送数据
```

**recover（恢复）：**

```go
// recover 只能在 defer 函数中使用
func safeDivide(a, b int) (result int, err error) {
    defer func() {
        if r := recover(); r != nil {
            err = fmt.Errorf("recovered: %v", r)
        }
    }()
    return a / b, nil  // b=0 时会 panic
}

result, err := safeDivide(10, 0)
// result=0, err="recovered: runtime error: integer divide by zero"
```

**panic vs error：**

| 特性 | panic | error |
|------|-------|-------|
| 严重程度 | 致命错误 | 可恢复错误 |
| 处理方式 | recover | if err != nil |
| 使用场景 | 程序无法继续 | 预期中的错误 |
| 推荐程度 | 尽量避免 | 推荐使用 |

**什么时候使用 panic：**

```go
// 1. 程序初始化失败（不可恢复）
func init() {
    if err := loadConfig(); err != nil {
        panic("failed to load config: " + err.Error())
    }
}

// 2. 不可能发生的错误
func mustParse(s string) int {
    v, err := strconv.Atoi(s)
    if err != nil {
        panic(err)  // 如果确定不会出错
    }
    return v
}

// 3. 测试中的断言
func TestSomething(t *testing.T) {
    // t.Fatal 内部使用 panic
}
```

**最佳实践：**

```go
// 错误处理（推荐）
func doSomething() error {
    result, err := riskyOperation()
    if err != nil {
        return fmt.Errorf("doSomething: %w", err)
    }
    // 使用 result...
    return nil
}

// panic + recover（不推荐，除非必要）
func doSomething() (err error) {
    defer func() {
        if r := recover(); r != nil {
            err = fmt.Errorf("panic: %v", r)
        }
    }()
    // 可能 panic 的代码...
    return nil
}
```

## 五、Go 的 init 函数

### 面试问题

init 函数是什么？和 main 函数有什么区别？多个包的 init 执行顺序是什么？

### 标准答案

**基本特性：**

```go
// init 函数在 main 之前自动执行
// 一个包可以有多个 init 函数
// 一个文件可以有多个 init 函数

// a.go
func init() {
    fmt.Println("a.go init 1")
}

func init() {
    fmt.Println("a.go init 2")
}

// b.go
func init() {
    fmt.Println("b.go init")
}

// 执行顺序：按文件名排序，同一文件内按定义顺序
// a.go init 1 → a.go init 2 → b.go init
```

**包级别的 init 执行顺序：**

```go
// main 包导入了 pkgA，pkgA 导入了 pkgB
// 执行顺序：pkgB.init() → pkgA.init() → main.init() → main.main()

// 依赖图：
// main → pkgA → pkgB
// 执行顺序：pkgB → pkgA → main
```

**init vs main：**

| 特性 | init | main |
|------|------|------|
| 每个包 | 可以有多个 | 只有 main 包有 |
| 自动执行 | 是 | 是 |
| 执行顺序 | main 之前 | 最后 |
| 用途 | 初始化 | 程序入口 |

**常见用法：**

```go
// 1. 初始化全局变量
var db *sql.DB

func init() {
    var err error
    db, err = sql.Open("mysql", "dsn")
    if err != nil {
        panic(err)
    }
}

// 2. 注册驱动
import (
    _ "github.com/go-sql-driver/mysql"  // 匿名导入，触发 init
)

// 3. 验证配置
func init() {
    if os.Getenv("DATABASE_URL") == "" {
        panic("DATABASE_URL is required")
    }
}
```

**面试加分点：** init 函数不能被显式调用。测试时 init 也会执行，这可能导致测试环境问题。推荐使用显式的初始化函数。
