## 一、Go 的常用标准库

### 面试问题

Go 有哪些常用的标准库？请举例说明它们的用途。

### 标准答案

**fmt — 格式化：**

```go
import "fmt"

// 输出
fmt.Println("hello", "world")
fmt.Printf("name: %s, age: %d\n", "Alice", 25)
fmt.Fprintf(os.Stderr, "error: %v\n", err)

// 格式化动词
// %v  默认格式
// %+v 带字段名的结构体
// %#v Go 语法表示
// %T  类型
// %d  十进制整数
// %f  浮点数
// %s  字符串
// %q  带引号的字符串
// %x  十六进制
// %p  指针
// %w  包装错误（fmt.Errorf）

// 字符串格式化
s := fmt.Sprintf("hello %s", "world")
```

**io / io/ioutil：**

```go
import (
    "io"
    "os"
)

// 读取所有内容（Go 1.16+ 推荐 os.ReadFile）
data, err := os.ReadFile("file.txt")

// 写入文件（Go 1.16+ 推荐 os.WriteFile）
err := os.WriteFile("file.txt", []byte("hello"), 0644)

// 流式复制
src, _ := os.Open("src.txt")
defer src.Close()
dst, _ := os.Create("dst.txt")
defer dst.Close()
n, err := io.Copy(dst, src)

// 读取指定字节
buf := make([]byte, 1024)
n, err := src.Read(buf)

// 带缓冲的读取
reader := bufio.NewReader(src)
line, err := reader.ReadString('\n')

// 带缓冲的写入
writer := bufio.NewDst(dst)
writer.WriteString("hello")
writer.Flush()
```

**net/http：**

```go
import (
    "net/http"
    "encoding/json"
)

    log.Fatal(err)

    "application/json",
    bytes.NewBuffer(jsonData))

    },

req.Header.Set("Authorization", "Bearer token")
resp, err := client.Do(req)
```

**encoding/json：**

```go
import "encoding/json"

// 序列化
type User struct {
    Name  string `json:"name"`
    Age   int    `json:"age,omitempty"`
    Email string `json:"email,omitempty"`
}

user := User{Name: "Alice", Age: 25}
data, err := json.Marshal(user)
// {"name":"Alice","age":25}

// 美化输出
data, err := json.MarshalIndent(user, "", "  ")

// 反序列化
var u User
err := json.Unmarshal(data, &u)

// 解码到 map
var m map[string]interface{}
err := json.Unmarshal(data, &m)

// 流式解码
decoder := json.NewDecoder(resp.Body)
var u User
err := decoder.Decode(&u)

// 流式编码
encoder := json.NewEncoder(w)
err := encoder.Encode(user)
```

**database/sql：**

```go
import (
    "database/sql"
    _ "github.com/go-sql-driver/mysql"
)

// 连接数据库
db, err := sql.Open("mysql", "user:password@tcp(localhost:3306)/dbname")
if err != nil {
    log.Fatal(err)
}
defer db.Close()

// 设置连接池
db.SetMaxOpenConns(25)
db.SetMaxIdleConns(5)
db.SetConnMaxLifetime(5 * time.Minute)

// 查询单行
var name string
err := db.QueryRow("SELECT name FROM users WHERE id = ?", 1).Scan(&name)

// 查询多行
rows, err := db.Query("SELECT name, age FROM users WHERE age > ?", 18)
if err != nil {
    log.Fatal(err)
}
defer rows.Close()

for rows.Next() {
    var name string
    var age int
    if err := rows.Scan(&name, &age); err != nil {
        log.Fatal(err)
    }
    fmt.Println(name, age)
}

if err := rows.Err(); err != nil {
    log.Fatal(err)
}

// 插入
result, err := db.Exec("INSERT INTO users (name, age) VALUES (?, ?)", "Alice", 25)
id, _ := result.LastInsertId()
affected, _ := result.RowsAffected()

// 预编译语句
stmt, err := db.Prepare("INSERT INTO users (name, age) VALUES (?, ?)")
defer stmt.Close()
stmt.Exec("Alice", 25)
stmt.Exec("Bob", 30)

// 事务
tx, err := db.Begin()
if err != nil {
    log.Fatal(err)
}
_, err = tx.Exec("UPDATE accounts SET balance = balance - ? WHERE id = ?", 100, 1)
if err != nil {
    tx.Rollback()
    log.Fatal(err)
}
_, err = tx.Exec("UPDATE accounts SET balance = balance + ? WHERE id = ?", 100, 2)
if err != nil {
    tx.Rollback()
    log.Fatal(err)
}
if err := tx.Commit(); err != nil {
    log.Fatal(err)
}
```

**sort：**

```go
import "sort"

// 基本排序
ints := []int{3, 1, 4, 1, 5, 9}
sort.Ints(ints)

strings := []string{"banana", "apple", "cherry"}
sort.Strings(strings)

// 自定义排序
type ByAge []User

func (a ByAge) Len() int           { return len(a) }
func (a ByAge) Less(i, j int) bool { return a[i].Age < a[j].Age }
func (a ByAge) Swap(i, j int)      { a[i], a[j] = a[j], a[i] }

sort.Sort(ByAge(users))

// 使用 sort.Slice（更简洁）
sort.Slice(users, func(i, j int) bool {
    return users[i].Age < users[j].Age
})

// 二分查找
sort.SearchInts(ints, 4)  // 返回索引
```

## 二、Go 的测试

### 面试问题

如何编写 Go 测试？有哪些测试类型？如何进行基准测试？

### 标准答案

**基本测试：**

```go
// calculator_test.go
package calculator

import "testing"

func TestAdd(t *testing.T) {
    result := Add(1, 2)
    if result != 3 {
        t.Errorf("Add(1, 2) = %d; want 3", result)
    }
}

// 表驱动测试
func TestAdd(t *testing.T) {
    tests := []struct {
        name     string
        a, b     int
        expected int
    }{
        {"正数", 1, 2, 3},
        {"负数", -1, -2, -3},
        {"零", 0, 0, 0},
        {"混合", -1, 2, 1},
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            result := Add(tt.a, tt.b)
            if result != tt.expected {
                t.Errorf("Add(%d, %d) = %d; want %d",
                    tt.a, tt.b, result, tt.expected)
            }
        })
    }
}
```

**测试辅助函数：**

```go
func TestSomething(t *testing.T) {
    // 设置和清理
    tmpDir := t.TempDir()

    // 跳过测试
    if testing.Short() {
        t.Skip("skipping test in short mode")
    }

    // 并行测试
    t.Parallel()

    // 失败后继续
    t.Fail()      // 标记失败，继续执行
    t.FailNow()   // 标记失败，停止当前测试
    t.Fatal("x")  // 打印消息并 FailNow
    t.Errorf("x") // 打印消息并 Fail
}
```

**测试夹具（TestMain）：**

```go
func TestMain(m *testing.M) {
    // 设置
    setup()

    // 运行测试
    code := m.Run()

    // 清理
    teardown()

    os.Exit(code)
}
```

**Mock 和 Stub：**

```go
// 接口定义
type UserRepository interface {
    FindByID(id int) (*User, error)
}

// Mock 实现
type MockUserRepo struct {
    users map[int]*User
}

func (m *MockUserRepo) FindByID(id int) (*User, error) {
    user, ok := m.users[id]
    if !ok {
        return nil, errors.New("not found")
    }
    return user, nil
}

// 测试中使用
func TestUserService(t *testing.T) {
    mockRepo := &MockUserRepo{
        users: map[int]*User{
            1: {ID: 1, Name: "Alice"},
        },
    }
    svc := NewUserService(mockRepo)

    user, err := svc.GetUser(1)
    if err != nil {
        t.Fatal(err)
    }
    if user.Name != "Alice" {
        t.Errorf("expected Alice, got %s", user.Name)
    }
}

// 使用 gomock（推荐）
//go:generate mockgen -source=user.go -destination=mock_user.go -package=service

func TestUserService(t *testing.T) {
    ctrl := gomock.NewController(t)
    defer ctrl.Finish()

    mockRepo := NewMockUserRepository(ctrl)
    mockRepo.EXPECT().FindByID(1).Return(&User{ID: 1, Name: "Alice"}, nil)

    svc := NewUserService(mockRepo)
    user, _ := svc.GetUser(1)
    if user.Name != "Alice" {
        t.Errorf("expected Alice, got %s", user.Name)
    }
}
```

## 三、Go 的代码规范

### 面试问题

Go 有哪些代码规范？如何保证代码质量？

### 标准答案

**gofmt 和 goimports：**

```bash
# gofmt：官方格式化工具（自动格式化）
gofmt -w .

# goimports：格式化 + 管理 import
go install golang.org/x/tools/cmd/goimports@latest
goimports -w .
```

**go vet：**

```go
// 静态分析工具
go vet ./...

// 检查内容：
// - printf 格式化参数不匹配
// - 不可达代码
// - 错误的锁使用
// - 等等
```

**golangci-lint：**

```bash
# 安装
go install github.com/golangci/golangci-lint/cmd/golangci-lint@latest

# 运行
golangci-lint run

# .golangci.yml 配置
linters:
  enable:
    - errcheck
    - gosimple
    - govet
    - ineffassign
    - staticcheck
    - unused
    - gocritic
    - gofmt
    - goimports
    - misspell
    - unconvert
```

**命名规范：**

```go
// 包名：小写，简短，无下划线
package httputil  // 好
package http_util // 不好

// 变量和函数：驼峰式
userName  // 好
user_name // 不好

// 常量：驼峰式或全大写下划线
MaxRetries  // 好
MAX_RETRIES // 也可以

// 接口：动词或形容词
type Reader interface { ... }  // 好
type Readable interface { ... } // 好
type IReader interface { ... } // 不好（Go 没有 I 前缀）

// 首字母缩略词全大写或全小写
HTTPClient  // 好
HttpClient  // 不好
xmlParser   // 好
XMLParser   // 也可以
```

**注释规范：**

```go
// Package httputil 提供 HTTP 工具函数。
package httputil

// DoRequest 发送 HTTP 请求并返回响应。
// 如果请求失败，返回错误。
func DoRequest(req *http.Request) (*http.Response, error) {
    // ...
}

// 导出的标识符必须有注释
// 注释以标识符名开头
// 以句号结尾
```

**错误处理规范：**

```go
// 不要忽略错误
result, err := doSomething()
if err != nil {
    return err
}

// 提供上下文
if err != nil {
    return fmt.Errorf("failed to process user %d: %w", id, err)
}

// 不要使用 panic 处理预期错误
// 不要使用 os.Exit（除了 main 函数）
```

## 四、Go 的性能优化

### 面试问题

如何优化 Go 程序的性能？有哪些技巧？

### 标准答案

**1. 减少内存分配：**

```go
// 预分配 slice
s := make([]int, 0, 1000)

// 复用 buffer
var buf bytes.Buffer
buf.Reset()

// 使用 sync.Pool
var pool = sync.Pool{
    New: func() interface{} {
        return new(bytes.Buffer)
    },
}

// 使用值类型而非指针
type Point struct {
    X, Y float64
}
// 比 *Point 更少堆分配
```

**2. 优化字符串操作：**

```go
// 使用 strings.Builder
var builder strings.Builder
for i := 0; i < 1000; i++ {
    builder.WriteString("hello")
}
s := builder.String()

// 使用 bytes.Buffer
var buf bytes.Buffer
for i := 0; i < 1000; i++ {
    buf.WriteString("hello")
}
s := buf.String()
```

**3. 优化 map 操作：**

```go
// 预分配 map
m := make(map[string]int, 1000)

// 使用 int key 而非 string（如果可能）
m := make(map[int]int)
// 比 map[string]int 更快
```

**4. 优化并发：**

```go
// 控制 goroutine 数量
sem := make(chan struct{}, 10)
for i := 0; i < 1000; i++ {
    sem <- struct{}{}
    go func(i int) {
        defer func() { <-sem }()
        doWork(i)
    }(i)
}

// 使用 worker pool
jobs := make(chan Job, 100)
for i := 0; i < 10; i++ {
    go worker(jobs)
}

// 使用 errgroup
g, ctx := errgroup.WithContext(ctx)
g.SetLimit(10)
for i := 0; i < 1000; i++ {
    i := i
    g.Go(func() error {
        return doWork(ctx, i)
    })
}
```

**5. 避免不必要的 interface{}：**

```go
// 不好：使用 interface{}
func process(v interface{}) {
    // 类型断言有开销
}

// 好：使用泛型（Go 1.18+）
func process[T any](v T) {
    // 无额外开销
}
```

**6. 使用编译器优化：**

```go
// 内联小函数
func add(a, b int) int {
    return a + b
}

// 使用 -ldflags 减小二进制体积
go build -ldflags="-s -w"

// 使用 -race 检测竞态
go build -race
go test -race
```

**7. 性能分析工具：**

```go
// pprof
go tool pprof http://localhost:6060/debug/pprof/profile

// trace
go tool trace trace.out

// benchmark
go test -bench=. -benchmem -cpuprofile=cpu.prof
go tool pprof cpu.prof
```

**面试加分点：** 优化之前先用 pprof 找到瓶颈，不要凭感觉优化。过早优化是万恶之源。
