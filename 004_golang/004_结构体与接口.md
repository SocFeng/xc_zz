# Go 结构体与接口

---

## 一、Go 的结构体

### 面试问题

Go 的结构体是什么？方法的值接收者和指针接收者有什么区别？

### 标准答案

**基本用法：**

```go
type User struct {
    Name  string
    Age   int
    Email string
}

// 创建
u := User{Name: "Alice", Age: 25}
u := User{"Alice", 25, "alice@example.com"}  // 按顺序
u := User{Name: "Alice"}  // 部分字段，其余为零值
u := new(User)  // *User，字段为零值

// 访问
fmt.Println(u.Name)
u.Name = "Bob"

// 匿名字段（嵌入）
type Admin struct {
    User        // 嵌入 User（不是继承）
    Level int
}

a := Admin{User: User{Name: "Alice"}, Level: 1}
fmt.Println(a.Name)  // 直接访问 User 的字段
fmt.Println(a.User.Name)  // 也可以显式访问
```

**方法：**

```go
// 值接收者
func (u User) GetInfo() string {
    return fmt.Sprintf("%s, %d岁", u.Name, u.Age)
}

// 指针接收者
func (u *User) SetName(name string) {
    u.Name = name
}

// 调用
u := User{Name: "Alice"}
u.GetInfo()     // 值接收者
u.SetName("Bob") // 指针接收者（Go 自动取地址）
(&u).SetName("Bob")  // 等价
```

**值接收者 vs 指针接收者：**

| 特性 | 值接收者 | 指针接收者 |
|------|---------|-----------|
| 修改原对象 | 不能 | 能 |
| 接收 nil | 不能 | 能 |
| 并发安全 | 是（拷贝） | 否 |
| 性能 | 大结构体有拷贝开销 | 无拷贝 |

**选择原则：**

```go
// 使用指针接收者的情况：
// 1. 需要修改接收者
func (u *User) SetName(name string) { u.Name = name }

// 2. 接收者是大结构体（避免拷贝）
func (s *LargeStruct) Process() { ... }

// 3. 结构体包含不能被拷贝的字段（sync.Mutex）
type SafeCounter struct {
    mu    sync.Mutex
    count int
}
func (c *SafeCounter) Increment() {
    c.mu.Lock()
    defer c.mu.Unlock()
    c.count++
}

// 4. 一致性：一个类型的方法应该统一使用值或指针接收者
// 推荐：如果有一个方法需要指针接收者，所有方法都用指针接收者

// 使用值接收者的情况：
// 1. 不需要修改接收者
// 2. 接收者是小的值类型（int, bool, string, 小 struct）
// 3. 类型是不可变的
// 4. 类型的方法不包含需要修改状态的方法
```

**结构体标签（Tag）：**

```go
type User struct {
    Name  string `json:"name" db:"name" validate:"required"`
    Age   int    `json:"age" db:"age" validate:"gte=0,lte=130"`
    Email string `json:"email,omitempty" db:"email"`
}

// 使用反射读取 tag
t := reflect.TypeOf(User{})
field, _ := t.FieldByName("Name")
fmt.Println(field.Tag.Get("json"))     // "name"
fmt.Println(field.Tag.Get("db"))       // "name"
fmt.Println(field.Tag.Get("validate")) // "required"

// JSON 序列化
import "encoding/json"
u := User{Name: "Alice", Age: 25}
data, _ := json.Marshal(u)
// {"name":"Alice","age":25}
```

**结构体的比较：**

```go
// 结构体可以比较（所有字段都可比较时）
type Point struct {
    X, Y int
}
p1 := Point{1, 2}
p2 := Point{1, 2}
fmt.Println(p1 == p2)  // true

// 包含不可比较字段（slice, map, func）时不能比较
type Data struct {
    Values []int
}
// d1 == d2  // 编译错误
```

## 二、Go 的接口

### 面试问题

Go 的接口是什么？什么是鸭子类型？空接口和类型断言是什么？

### 标准答案

**接口的定义：**

```go
// 接口是一组方法的集合
type Writer interface {
    Write(p []byte) (n int, err error)
}

type Reader interface {
    Read(p []byte) (n int, err error)
}

// 接口组合
type ReadWriter interface {
    Reader
    Writer
}

// 实现接口（隐式实现，不需要声明 implements）
type FileWriter struct {
    file *os.File
}

func (fw FileWriter) Write(p []byte) (n int, err error) {
    return fw.file.Write(p)
}
// FileWriter 自动实现了 Writer 接口
```

**鸭子类型：**

```go
// "如果它走起来像鸭子，叫起来像鸭子，那它就是鸭子"
// Go 的接口是鸭子类型的静态实现

type Animal interface {
    Speak() string
}

type Dog struct{}
func (d Dog) Speak() string { return "Woof!" }

type Cat struct{}
func (c Cat) Speak() string { return "Meow!" }

func makeSound(a Animal) {
    fmt.Println(a.Speak())
}

makeSound(Dog{})  // Woof!
makeSound(Cat{})  // Meow!
// Dog 和 Cat 都没有声明实现 Animal，但它们有 Speak() 方法
```

**接口作为函数参数：**

```go
// 接口越大，抽象越弱
// 接口越小，抽象越强

// Go 推崇小接口
type Stringer interface {
    String() string
}

type Reader interface {
    Read(p []byte) (n int, err error)
}

// 标准库中的经典小接口
// io.Reader, io.Writer, fmt.Stringer, error
```

**空接口 `interface{}`：**

```go
// 空接口没有任何方法，所有类型都实现了空接口
func describe(i interface{}) {
    fmt.Printf("(%v, %T)\n", i, i)
}

describe(42)       // (42, int)
describe("hello")  // (hello, string)
describe(true)     // (true, bool)

// Go 1.18+ 可以用 any 替代 interface{}
func describe(i any) {
    fmt.Printf("(%v, %T)\n", i, i)
}

// 空接口的用途
// 1. 接收任意类型
var m map[string]interface{} = make(map[string]interface{})
m["name"] = "Alice"
m["age"] = 25

// 2. 泛型之前的数据容器
type Container struct {
    items []interface{}
}

func (c *Container) Add(item interface{}) {
    c.items = append(c.items, item)
}
```

**类型断言和类型选择：**

```go
// 类型断言
var i interface{} = "hello"

s, ok := i.(string)  // 安全断言
if ok {
    fmt.Println(s)  // hello
}

s := i.(string)  // 非安全断言，失败会 panic

// 类型选择（type switch）
func describe(i interface{}) {
    switch v := i.(type) {
    case int:
        fmt.Printf("int: %d\n", v)
    case string:
        fmt.Printf("string: %s\n", v)
    case bool:
        fmt.Printf("bool: %t\n", v)
    case nil:
        fmt.Println("nil")
    default:
        fmt.Printf("unknown: %T\n", v)
    }
}
```

**接口的底层原理：**

```go
// 接口的底层结构（runtime/runtime2.go）
type iface struct {
    tab  *itab          // 类型信息
    data unsafe.Pointer // 数据指针
}

type eface struct {  // 空接口
    _type *_type
    data  unsafe.Pointer
}

// 接口值 = 类型信息 + 数据指针
var w io.Writer
// w 是 nil 接口（类型和数据都是 nil）

w = os.Stdout
// w 的类型是 *os.File，数据是 os.Stdout 的指针

w = new(bytes.Buffer)
// w 的类型是 *bytes.Buffer，数据是 Buffer 的指针
```

**接口的注意事项：**

```go
// 1. 接口值为 nil 时调用方法会 panic
var w io.Writer
w.Write([]byte("hello"))  // panic: nil pointer

// 2. 接口值不为 nil 但数据为 nil
var buf *bytes.Buffer
var w io.Writer = buf  // w 不是 nil！类型信息不为 nil
w.Write([]byte("hello"))  // panic: nil pointer

// 正确的 nil 检查
if w != nil && !reflect.ValueOf(w).IsNil() {
    w.Write([]byte("hello"))
}

// 3. 接口的比较
// 接口值相等 = 类型相同 AND 数据相等
var a io.Writer = bytes.NewBuffer(nil)
var b io.Writer = bytes.NewBuffer(nil)
fmt.Println(a == b)  // false（不同实例）

// 4. 接口不能包含未导出方法（跨包时）
// 包 A
type MyInterface interface {
    method()  // 小写，未导出
}
// 包 B 无法实现这个接口
```

**面试加分点：** Go 的接口是隐式实现的，这使得代码更加灵活。可以在不修改现有代码的情况下，让新类型实现已有接口。这被称为"鸭子类型"的静态版本。

## 三、Go 的组合（Composition）

### 面试问题

Go 没有继承，如何实现代码复用？组合和继承有什么区别？

### 标准答案

**结构体嵌入（Embedding）：**

```go
type Base struct {
    Id int
}

func (b *Base) Describe() string {
    return fmt.Sprintf("Base[%d]", b.Id)
}

type User struct {
    Base        // 嵌入 Base（不是继承）
    Name string
}

type Admin struct {
    User        // 嵌入 User
    Level int
}

// 使用
a := Admin{}
a.Id = 1           // 访问 Base 的字段
a.Name = "Alice"   // 访问 User 的字段
a.Level = 1        // 访问自己的字段
a.Describe()       // 调用 Base 的方法

// 等价于
a.User.Base.Describe()
```

**组合 vs 继承：**

| 特性 | 组合（Go） | 继承（Java/Python） |
|------|-----------|-------------------|
| 关系 | has-a | is-a |
| 耦合度 | 低 | 高 |
| 灵活性 | 高 | 低 |
| 多态 | 接口 | 虚函数 |
| 方法覆盖 | 不支持 | 支持 |

**接口组合：**

```go
type Reader interface {
    Read(p []byte) (n int, err error)
}

type Writer interface {
    Write(p []byte) (n int, err error)
}

type ReadWriter interface {
    Reader
    Writer
}

// 任何实现了 Read 和 Write 的类型都自动实现 ReadWriter
type Buffer struct {
    data []byte
}

func (b *Buffer) Read(p []byte) (n int, err error) {
    // ...
}

func (b *Buffer) Write(p []byte) (n int, err error) {
    // ...
}
// Buffer 实现了 ReadWriter
```

**设计原则：**

```go
// Go 推崇 "Accept interfaces, return structs"
// 接受接口，返回结构体

// 好的设计
func Process(r io.Reader) error {
    // 使用接口，灵活
    data, err := io.ReadAll(r)
    // ...
}

// 不好的设计
func Process(f *os.File) error {
    // 使用具体类型，不灵活
    // 无法传入 bytes.Buffer, net.Conn 等
}

// 工厂函数返回具体类型
func NewBuffer() *Buffer {
    return &Buffer{}
}
```

## 四、Go 的常用标准接口

### 面试问题

Go 标准库中有哪些常用接口？

### 标准答案

```go
// 1. error 接口
type error interface {
    Error() string
}

// 2. fmt.Stringer
type Stringer interface {
    String() string
}

// 3. io.Reader
type Reader interface {
    Read(p []byte) (n int, err error)
}

// 4. io.Writer
type Writer interface {
    Write(p []byte) (n int, err error)
}

// 5. io.Closer
type Closer interface {
    Close() error
}

// 6. io.ReadWriter
type ReadWriter interface {
    Reader
    Writer
}

// 7. io.ReadCloser
type ReadCloser interface {
    Reader
    Closer
}

// 8. sort.Interface
type Interface interface {
    Len() int
    Less(i, j int) bool
    Swap(i, j int)
}

// 9. sort.Sort
// sort.Sort(MySlice)  // 实现了 sort.Interface 的类型

// 10. database/sql 中的接口
type Scanner interface {
    Scan(dest ...interface{}) error
}

// 11. encoding.BinaryMarshaler / BinaryUnmarshaler
type BinaryMarshaler interface {
    MarshalBinary() (data []byte, err error)
}

// 12. encoding.TextMarshaler / TextUnmarshaler
type TextMarshaler interface {
    MarshalText() (text []byte, err error)
}

// 13. json.Marshaler / json.Unmarshaler
type Marshaler interface {
    MarshalJSON() ([]byte, error)
}

// 14. http.Handler
type Handler interface {
    ServeHTTP(ResponseWriter, *Request)
}

// 15. http.HandlerFunc
type HandlerFunc func(ResponseWriter, *Request)
```
