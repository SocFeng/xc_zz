# Go 错误处理

---

## 一、Go 的 error 接口

### 面试问题

Go 的错误处理机制是什么？和其他语言有什么区别？

### 标准答案

**error 接口：**

```go
// error 是一个内置接口
type error interface {
    Error() string
}

// 创建 error 的方式
// 1. errors.New
err := errors.New("something went wrong")

// 2. fmt.Errorf
err := fmt.Errorf("user %s not found", name)

// 3. 自定义 error 类型
type NotFoundError struct {
    Resource string
    ID       int
}

func (e *NotFoundError) Error() string {
    return fmt.Sprintf("%s with id %d not found", e.Resource, e.ID)
}

err := &NotFoundError{Resource: "User", ID: 42}
```

**错误处理模式：**

```go
// Go 推崇显式错误处理
result, err := doSomething()
if err != nil {
    return err
}

// 多返回值模式
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("division by zero")
    }
    return a / b, nil
}

// 错误链
func processUser(id int) error {
    user, err := findUser(id)
    if err != nil {
        return fmt.Errorf("processUser: %w", err)  // %w 包装错误
    }
    // ...
}
```

**Go 错误处理 vs 其他语言：**

| 特性 | Go | Java/Python |
|------|-----|-------------|
| 机制 | 返回值（error） | 异常（try-catch） |
| 显式性 | 必须显式处理 | 可以忽略 |
| 性能 | 无额外开销 | 有栈展开开销 |
| 控制流 | 线性 | 跳转 |
| 强制性 | 编译器不强制 | 可以不捕获 |

## 二、Go 的错误包装（Wrapping）

### 面试问题

如何包装错误？如何判断错误链中的特定错误？

### 标准答案

**Go 1.13+ 错误包装：**

```go
// %w 包装错误（保留原始错误）
func readConfig(filename string) error {
    data, err := os.ReadFile(filename)
    if err != nil {
        return fmt.Errorf("readConfig: %w", err)
    }
    // ...
}

// errors.Is：判断错误链中是否包含特定错误
if errors.Is(err, os.ErrNotExist) {
    fmt.Println("file not found")
}

// errors.As：从错误链中提取特定类型的错误
var pathErr *os.PathError
if errors.As(err, &pathErr) {
    fmt.Println("path:", pathErr.Path)
}

// errors.Unwrap：获取被包装的错误
unwrapped := errors.Unwrap(err)
```

**自定义可包装的错误：**

```go
type MyError struct {
    Code    int
    Message string
    Err     error  // 被包装的错误
}

func (e *MyError) Error() string {
    if e.Err != nil {
        return fmt.Sprintf("[%d] %s: %v", e.Code, e.Message, e.Err)
    }
    return fmt.Sprintf("[%d] %s", e.Code, e.Message)
}

func (e *MyError) Unwrap() error {
    return e.Err
}

// 使用
err := &MyError{
    Code:    500,
    Message: "internal error",
    Err:     errors.New("database connection failed"),
}

// errors.Is 会沿着错误链查找
if errors.Is(err, errors.New("database connection failed")) {
    // 这个不会匹配，因为 errors.New 创建的是新实例
}

// errors.As 可以提取自定义类型
var myErr *MyError
if errors.As(err, &myErr) {
    fmt.Println(myErr.Code)  // 500
}
```

**错误判断的几种方式：**

```go
// 1. 直接比较（哨兵错误）
if err == io.EOF {
    // ...
}

// 2. errors.Is（推荐，支持错误链）
if errors.Is(err, io.EOF) {
    // ...
}

// 3. 类型断言
if pathErr, ok := err.(*os.PathError); ok {
    // ...
}

// 4. errors.As（推荐，支持错误链）
var pathErr *os.PathError
if errors.As(err, &pathErr) {
    // ...
}
```

## 三、Go 的错误处理最佳实践

### 面试问题

Go 的错误处理有哪些最佳实践？如何设计好的错误处理？

### 标准答案

**1. 错误应该被处理或传播：**

```go
// 不好：忽略错误
doSomething()

// 好：处理或传播
if err := doSomething(); err != nil {
    return err
}
```

**2. 错误应该提供上下文：**

```go
// 不好：没有上下文
if err != nil {
    return err
}

// 好：提供上下文
if err != nil {
    return fmt.Errorf("failed to process user %d: %w", id, err)
}
```

**3. 使用哨兵错误：**

```go
// 定义包级别的哨兵错误
var (
    ErrNotFound  = errors.New("not found")
    ErrForbidden = errors.New("forbidden")
)

func GetUser(id int) (*User, error) {
    user, ok := db[id]
    if !ok {
        return nil, ErrNotFound
    }
    return user, nil
}

// 调用者可以判断具体错误
user, err := GetUser(42)
if errors.Is(err, ErrNotFound) {
    // 处理未找到
}
```

**4. 自定义错误类型：**

```go
type ValidationError struct {
    Field   string
    Message string
}

func (e *ValidationError) Error() string {
    return fmt.Sprintf("validation error: %s - %s", e.Field, e.Message)
}

func ValidateUser(u *User) error {
    if u.Name == "" {
        return &ValidationError{Field: "name", Message: "required"}
    }
    if u.Age < 0 {
        return &ValidationError{Field: "age", Message: "must be positive"}
    }
    return nil
}

// 调用者可以提取详细信息
err := ValidateUser(user)
var validationErr *ValidationError
if errors.As(err, &validationErr) {
    fmt.Println(validationErr.Field)   // "name"
    fmt.Println(validationErr.Message) // "required"
}
```

**5. 不要过度使用 panic：**

```go
// 不好：使用 panic 处理预期错误
func divide(a, b int) int {
    if b == 0 {
        panic("division by zero")  // 不要这样
    }
    return a / b
}

// 好：使用 error
func divide(a, b int) (int, error) {
    if b == 0 {
        return 0, errors.New("division by zero")
    }
    return a / b, nil
}

// 适合使用 panic 的场景：
// 1. 程序初始化失败
// 2. 不可能发生的错误
// 3. 测试中断言
```

**6. 错误处理与 defer：**

```go
func processFile(filename string) (err error) {
    f, err := os.Open(filename)
    if err != nil {
        return err
    }
    defer func() {
        if closeErr := f.Close(); closeErr != nil {
            err = fmt.Errorf("close error: %w (original: %v)", closeErr, err)
        }
    }()
    // 处理文件...
    return nil
}
```

**7. 错误分组：**

```go
// 多个错误的处理
type MultiError struct {
    Errors []error
}

func (me *MultiError) Error() string {
    msgs := make([]string, len(me.Errors))
    for i, err := range me.Errors {
        msgs[i] = err.Error()
    }
    return strings.Join(msgs, "; ")
}

func (me *MultiError) Add(err error) {
    if err != nil {
        me.Errors = append(me.Errors, err)
    }
}

func (me *MultiError) HasErrors() bool {
    return len(me.Errors) > 0
}

// 使用
var me MultiError
me.Add(validateName(u.Name))
me.Add(validateAge(u.Age))
me.Add(validateEmail(u.Email))
if me.HasErrors() {
    return &me
}
```

## 四、Go 的错误处理与第三方库

### 面试问题

Go 有哪些错误处理的第三方库？如何选择？

### 标准答案

**1. pkg/errors：**

```go
import "github.com/pkg/errors"

// 包装错误并保留堆栈
func process() error {
    err := doSomething()
    if err != nil {
        return errors.Wrap(err, "process failed")
    }
    return nil
}

// 获取堆栈信息
type stackTracer interface {
    StackTrace() errors.StackTrace
}

err := process()
if st, ok := err.(stackTracer); ok {
    fmt.Printf("%+v\n", st.StackTrace())
}

// errors.Cause 获取原始错误
if errors.Cause(err) == ErrNotFound {
    // ...
}
```

**2. Go 1.13+ 内置：**

```go
// 推荐使用标准库
// %w 包装错误
return fmt.Errorf("process failed: %w", err)

// errors.Is 判断
if errors.Is(err, os.ErrNotExist) {
    // ...
}

// errors.As 提取
var pathErr *os.PathError
if errors.As(err, &pathErr) {
    // ...
}
```

**3. 错误处理的选择：**

| 库 | 特点 | 推荐度 |
|----|------|--------|
| 标准库 errors | 官方支持，无依赖 | 推荐 |
| pkg/errors | 堆栈跟踪，兼容性好 | 可用 |
| Go 1.13+ fmt.Errorf | 包装错误，标准方式 | 推荐 |
