## 一、Python 的文件操作

### 面试问题

Python 中如何进行文件操作？有哪些打开模式？`pathlib` 是什么？

### 标准答案

**基本文件操作：**

```python
# 1. 基本读写
with open("test.txt", "r", encoding="utf-8") as f:
    content = f.read()        # 读取全部内容
    # content = f.read(100)   # 读取前 100 个字符

with open("test.txt", "r", encoding="utf-8") as f:
    lines = f.readlines()     # 读取所有行，返回列表
    # line = f.readline()     # 读取一行

with open("output.txt", "w", encoding="utf-8") as f:
    f.write("hello\n")        # 写入字符串
    f.writelines(["a\n", "b\n"])  # 写入多行

# 2. 逐行读取（内存友好）
with open("large.txt", "r") as f:
    for line in f:            # 文件对象本身是迭代器
        process(line.strip())
```

**打开模式：**

| 模式 | 说明 | 文件不存在 | 已有文件 |
|------|------|-----------|---------|
| `r` | 只读（默认） | 报错 | 读取 |
| `w` | 写入 | 创建 | 清空 |
| `a` | 追加 | 创建 | 追加 |
| `x` | 创建（已存在则报错） | 创建 | 报错 |
| `rb` | 二进制只读 | 报错 | 读取 |
| `wb` | 二进制写入 | 创建 | 清空 |
| `r+` | 读写 | 报错 | 读写 |
| `w+` | 读写 | 创建 | 清空 |

**文件指针操作：**

```python
with open("test.txt", "r") as f:
    print(f.tell())      # 0，当前位置
    f.seek(5)            # 移动到第 5 个字节
    print(f.tell())      # 5
    content = f.read()   # 从第 5 个字节开始读
```

**`pathlib` 模块（Python 3.4+）：**

```python
from pathlib import Path

# 创建路径
p = Path("/home/user/documents")
p = Path("home") / "user" / "documents"  # 使用 / 拼接

# 常用方法
p.exists()           # 是否存在
p.is_file()          # 是否是文件
p.is_dir()           # 是否是目录
p.name               # 文件名 "documents"
p.stem               # 不含后缀的文件名
p.suffix             # 后缀 ".txt"
p.parent             # 父目录
p.resolve()          # 绝对路径

# 文件操作
p = Path("test.txt")
p.write_text("hello", encoding="utf-8")
content = p.read_text(encoding="utf-8")
p.write_bytes(b"\x00\x01")
data = p.read_bytes()

# 目录操作
Path("new_dir").mkdir(parents=True, exist_ok=True)
Path("old.txt").rename("new.txt")
Path("file.txt").unlink()  # 删除文件

# 遍历目录
for f in Path(".").glob("*.py"):        # 当前目录
    print(f)

for f in Path(".").rglob("*.py"):       # 递归搜索
    print(f)

# 文件信息
stat = p.stat()
print(stat.st_size)      # 文件大小
print(stat.st_mtime)     # 修改时间
```

**`os.path` vs `pathlib`：**

```python
import os
from pathlib import Path

# 旧方式
path = "/home/user/test.txt"
print(os.path.basename(path))    # test.txt
print(os.path.dirname(path))     # /home/user
print(os.path.exists(path))      # False
print(os.path.splitext(path))    # ('/home/user/test', '.txt')

# 新方式（推荐）
p = Path("/home/user/test.txt")
print(p.name)                    # test.txt
print(p.parent)                  # /home/user
print(p.exists())                # False
print(p.suffix)                  # .txt
print(p.stem)                    # test
```

**遍历目录：**

```python
from pathlib import Path

# 遍历当前目录
for p in Path(".").iterdir():
    print(p)

# 递归查找所有 Python 文件
for p in Path(".").rglob("*.py"):
    print(p)

# glob 模式匹配
for p in Path(".").glob("*.py"):
    print(p)
```

## 二、Python 的 JSON 序列化

### 面试问题

Python 中如何进行 JSON 序列化和反序列化？如何处理自定义对象？

### 标准答案

**基本用法：**

```python
import json

# 序列化：Python 对象 → JSON 字符串
data = {"name": "Alice", "age": 25, "scores": [90, 85, 92]}
json_str = json.dumps(data, ensure_ascii=False, indent=2)
print(json_str)

# 写入文件
with open("data.json", "w", encoding="utf-8") as f:
    json.dump(data, f, ensure_ascii=False, indent=2)

# 反序列化：JSON 字符串 → Python 对象
data = json.loads(json_str)

# 从文件读取
with open("data.json", "r", encoding="utf-8") as f:
    data = json.load(f)
```

**JSON 与 Python 类型对应关系：**

| JSON | Python |
|------|--------|
| object | dict |
| array | list |
| string | str |
| number (int) | int |
| number (float) | float |
| true/false | True/False |
| null | None |

**处理自定义对象：**

```python
import json
from datetime import datetime

class User:
    def __init__(self, name, created_at):
        self.name = name
        self.created_at = created_at

# 方式一：自定义 encoder
class UserEncoder(json.JSONEncoder):
    def default(self, obj):
        if isinstance(obj, User):
            return {
                "__type__": "User",
                "name": obj.name,
                "created_at": obj.created_at.isoformat()
            }
        if isinstance(obj, datetime):
            return obj.isoformat()
        return super().default(obj)

user = User("Alice", datetime.now())
json_str = json.dumps(user, cls=UserEncoder)
print(json_str)

# 方式二：使用 default 参数
def user_to_dict(obj):
    if isinstance(obj, User):
        return {"name": obj.name, "created_at": obj.created_at.isoformat()}
    if isinstance(obj, datetime):
        return obj.isoformat()
    raise TypeError(f"Object of type {type(obj)} is not JSON serializable")

json_str = json.dumps(user, default=user_to_dict)

# 反序列化自定义对象
def dict_to_user(d):
    if "__type__" in d and d["__type__"] == "User":
        return User(d["name"], datetime.fromisoformat(d["created_at"]))
    return d

data = json.loads(json_str, object_hook=dict_to_user)
```

**常用参数：**

```python
json.dumps(data,
    ensure_ascii=False,   # 支持中文
    indent=2,             # 缩进
    sort_keys=True,       # 排序键
    separators=(',', ':'),  # 自定义分隔符
    default=str,          # 默认转换函数
)
```

## 三、Python 的 pickle 序列化

### 面试问题

pickle 和 json 有什么区别？pickle 有什么安全风险？

### 标准答案

**基本用法：**

```python
import pickle

# 序列化
data = {"name": "Alice", "scores": [90, 85]}
bytes_data = pickle.dumps(data)

# 写入文件
with open("data.pkl", "wb") as f:
    pickle.dump(data, f)

# 反序列化
data = pickle.loads(bytes_data)

# 从文件读取
with open("data.pkl", "rb") as f:
    data = pickle.load(f)
```

**pickle vs json：**

| 特性 | pickle | json |
|------|--------|------|
| 格式 | 二进制 | 文本 |
| 支持类型 | 几乎所有 Python 对象 | 有限（dict list str int float None bool） |
| 可读性 | 不可读 | 可读 |
| 语言通用性 | 仅 Python | 跨语言 |
| 安全性 | 不安全 | 安全 |
| 速度 | 较快 | 较慢 |

**pickle 的安全风险：**

```python
# 危险！不要反序列化不信任的数据
import pickle
import os

class Exploit:
    def __reduce__(self):
        # 反序列化时会执行 os.system("rm -rf /")
        return (os.system, ("rm -rf /",))

# 攻击者可以构造恶意 pickle 数据
# 当你 pickle.load() 时就会执行任意代码

# 安全的做法
# 1. 只反序列化自己序列化的数据
# 2. 使用 json 替代（如果可能）
# 3. 使用 hmac 签名验证数据完整性
import hmac

def safe_dump(data, key):
    raw = pickle.dumps(data)
    sig = hmac.new(key, raw, "sha256").digest()
    return raw + sig

def safe_load(data_with_sig, key):
    raw = data_with_sig[:-32]
    sig = data_with_sig[-32:]
    expected = hmac.new(key, raw, "sha256").digest()
    if not hmac.compare_digest(sig, expected):
        raise ValueError("数据被篡改")
    return pickle.loads(raw)
```

**pickle 的高级用法：**

```python
import pickle

# 自定义序列化行为
class MyClass:
    def __init__(self, data):
        self.data = data
        self.cache = {}  # 不需要序列化

    def __getstate__(self):
        """自定义序列化"""
        state = self.__dict__.copy()
        del state['cache']  # 排除 cache
        return state

    def __setstate__(self, state):
        """自定义反序列化"""
        self.__dict__.update(state)
        self.cache = {}  # 重新初始化 cache

# 深拷贝的原理
import copy
data = [1, 2, [3, 4]]
deep = copy.deepcopy(data)
# deepcopy 底层使用 pickle 的协议实现
```

## 四、Python 的 logging 模块

### 面试问题

Python 的 logging 模块是如何工作的？如何配置日志？

### 标准答案

**日志级别：**

| 级别 | 数值 | 说明 |
|------|------|------|
| DEBUG | 10 | 调试信息 |
| INFO | 20 | 一般信息 |
| WARNING | 30 | 警告（默认级别） |
| ERROR | 40 | 错误 |
| CRITICAL | 50 | 严重错误 |

**基本配置：**

```python
import logging

# 简单配置
logging.basicConfig(
    level=logging.DEBUG,
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s',
    filename='app.log',
    filemode='a'
)

logging.debug("调试信息")
logging.info("一般信息")
logging.warning("警告")
logging.error("错误")
logging.critical("严重错误")
```

**四大组件：**

```python
import logging

# 1. Logger：日志记录器
logger = logging.getLogger(__name__)
logger.setLevel(logging.DEBUG)

# 2. Handler：日志处理器
# 控制台处理器
console_handler = logging.StreamHandler()
console_handler.setLevel(logging.INFO)

# 文件处理器
file_handler = logging.FileHandler('app.log')
file_handler.setLevel(logging.DEBUG)

# 轮转文件处理器（按大小轮转）
from logging.handlers import RotatingFileHandler
rotating_handler = RotatingFileHandler(
    'app.log', maxBytes=1024*1024, backupCount=5
)

# 按时间轮转
from logging.handlers import TimedRotatingFileHandler
time_handler = TimedRotatingFileHandler(
    'app.log', when='midnight', interval=1, backupCount=30
)

# 3. Filter：日志过滤器
class CustomFilter(logging.Filter):
    def filter(self, record):
        return record.levelno >= logging.WARNING

# 4. Formatter：日志格式器
formatter = logging.Formatter(
    '%(asctime)s - %(name)s - %(levelname)s - %(message)s'
)

# 组装
console_handler.setFormatter(formatter)
file_handler.setFormatter(formatter)
logger.addHandler(console_handler)
logger.addHandler(file_handler)

logger.info("这条会显示在控制台和文件中")
```

**字典配置（推荐）：**

```python
import logging.config

LOGGING_CONFIG = {
    'version': 1,
    'disable_existing_loggers': False,
    'formatters': {
        'standard': {
            'format': '%(asctime)s - %(name)s - %(levelname)s - %(message)s'
        },
    },
    'handlers': {
        'console': {
            'class': 'logging.StreamHandler',
            'formatter': 'standard',
            'level': 'INFO',
        },
        'file': {
            'class': 'logging.handlers.RotatingFileHandler',
            'filename': 'app.log',
            'maxBytes': 1024*1024,
            'backupCount': 5,
            'formatter': 'standard',
            'level': 'DEBUG',
        },
    },
    'loggers': {
        '': {  # root logger
            'handlers': ['console', 'file'],
            'level': 'DEBUG',
        },
    },
}

logging.config.dictConfig(LOGGING_CONFIG)
logger = logging.getLogger(__name__)
```

**最佳实践：**

```python
# 使用模块名作为 Logger 名称
logger = logging.getLogger(__name__)

# 不要在模块级别添加 Handler（避免重复日志）
# 在应用入口处配置一次即可

# 使用 %s 而不是 f-string（延迟格式化）
logger.debug("User %s logged in from %s", user_id, ip_address)
# 而不是
logger.debug(f"User {user_id} logged in from {ip_address}")  # 即使不输出也会格式化

# 生产环境使用 WARNING 或以上级别
# 开发环境使用 DEBUG
```

**面试加分点：** `logging.getLogger(__name__)` 使用模块名作为 Logger 名称，可以形成层次结构，方便按模块控制日志级别。
