## 一、Python 的数据类型及其特性

### 面试问题

Python 有哪些数据类型？可变类型和不可变类型分别有哪些？它们有什么区别？

### 标准答案

Python 的数据类型分为**不可变类型**和**可变类型**两大类。

**不可变类型（Immutable）：**

| 类型 | 说明 | 示例 |
|------|------|------|
| `int` | 整数，任意精度，不受位数限制 | `a = 42` |
| `float` | 浮点数，双精度64位 | `b = 3.14` |
| `str` | 字符串，不可变的 Unicode 字符序列 | `s = "hello"` |
| `tuple` | 元组，不可变有序序列 | `t = (1, 2, 3)` |
| `frozenset` | 不可变集合 | `fs = frozenset([1, 2])` |
| `bytes` | 不可变字节序列 | `b = b"hello"` |

**可变类型（Mutable）：**

| 类型 | 说明 | 示例 |
|------|------|------|
| `list` | 列表，动态数组，支持增删改查 | `lst = [1, 2, 3]` |
| `dict` | 字典，哈希表实现，键值对存储 | `d = {"a": 1}` |
| `set` | 集合，哈希表实现，元素唯一 | `s = {1, 2, 3}` |
| `bytearray` | 可变字节序列 | `ba = bytearray(b"hello")` |

**核心区别：**

```python
# 不可变类型：修改值会创建新对象
a = "hello"
print(id(a))  # 140234866534128
a = a + " world"
print(id(a))  # 140234866534192  地址变了，是新对象

# 可变类型：修改值直接在原对象上修改
lst = [1, 2, 3]
print(id(lst))  # 140234866345024
lst.append(4)
print(id(lst))  # 140234866345024  地址没变，是同一个对象
```

**面试加分点：** 不可变类型可以作为字典的 key 和集合的元素（因为哈希值不变），可变类型不行。

## 二、深拷贝与浅拷贝

### 面试问题

Python 中深拷贝和浅拷贝有什么区别？分别在什么场景下使用？

### 标准答案

**浅拷贝（Shallow Copy）：** 只复制对象本身，不复制内部引用的子对象。子对象仍然指向原来的引用。

```python
import copy

original = [[1, 2], [3, 4]]
shallow = copy.copy(original)

shallow[0][0] = 99
print(original)  # [[99, 2], [3, 4]]  原对象也被修改了！
```

浅拷贝的实现方式：
- `copy.copy()`
- `list[:]` 切片
- `dict.copy()` / `dict()`
- `list()` 构造函数

**深拷贝（Deep Copy）：** 递归复制对象及其所有子对象，完全独立。

```python
import copy

original = [[1, 2], [3, 4]]
deep = copy.deepcopy(original)

deep[0][0] = 99
print(original)  # [[1, 2], [3, 4]]  原对象不受影响
```

**不可变类型的情况：**

```python
a = (1, [2, 3])
b = copy.copy(a)
c = copy.deepcopy(a)

# 不可变类型本身不会被修改，但如果包含可变元素
a[1].append(4)
print(b)  # (1, [2, 3, 4])  浅拷贝受影响
print(c)  # (1, [2, 3])     深拷贝不受影响
```

**面试加分点：** 纯不可变类型（int、str、不含可变元素的 tuple）赋值即引用，深浅拷贝无区别。

## 三、Python 的 GIL（全局解释器锁）

### 面试问题

什么是 GIL？它为什么存在？对程序有什么影响？如何绕过？

### 标准答案

**GIL（Global Interpreter Lock）** 是 CPython 解释器中的一把全局锁，同一时刻只有一个线程执行 Python 字节码。

**为什么存在：**

1. **内存管理不是线程安全的**：CPython 使用引用计数管理内存，引用计数的增减不是原子操作，多线程同时修改会导致内存泄漏或提前释放
2. **简化 CPython 实现**：GIL 让 CPython 的单线程性能更好，避免了大量的锁开销
3. **历史原因**：Python 诞生于 1991 年，当时多核 CPU 还不普及

**对程序的影响：**

```python
# CPU 密集型：多线程无法利用多核，甚至比单线程慢
import threading, time

def cpu_bound():
    total = 0
    for i in range(10**7):
        total += i

# 单线程
start = time.time()
cpu_bound()
cpu_bound()
print(f"单线程: {time.time() - start:.2f}s")  # 约 2s

# 多线程
start = time.time()
t1 = threading.Thread(target=cpu_bound)
t2 = threading.Thread(target=cpu_bound)
t1.start(); t2.start()
t1.join(); t2.join()
print(f"多线程: {time.time() - start:.2f}s")  # 约 2s，没有加速！
```

```python
# IO 密集型：线程在 IO 等待时释放 GIL，多线程仍然有效
import threading, requests

def io_bound(url):
    requests.get(url)

# 多线程访问多个 URL，明显比单线程快
```

**绕过 GIL 的方案：**

1. **多进程 multiprocessing**：每个进程有独立的 GIL
2. **C 扩展**：在 C 代码中释放 GIL（如 NumPy 的计算）
3. **其他解释器**：Jython、PyPy（PyPy 也有 GIL，但有 STM 分支）
4. **Python 3.13+**：实验性移除 GIL（`--disable-gil` 编译选项）

**面试加分点：** GIL 是 CPython 特有的，不是 Python 语言规范的一部分。Python 3.2+ 对 GIL 进行了优化，切换粒度从字节码指令改为时间片（默认 5ms）。

## 四、Python 的内存管理机制

### 面试问题

Python 是如何管理内存的？引用计数、标记清除、分代回收分别是什么？

### 标准答案

Python 的内存管理采用**三级机制**：

**1. 引用计数（Primary）：**

每个对象维护一个引用计数器，引用为 0 时立即回收。

```python
import sys

a = [1, 2, 3]
print(sys.getrefcount(a))  # 2（a 本身 + getrefcount 参数）

b = a  # 引用 +1
print(sys.getrefcount(a))  # 3

del b  # 引用 -1
print(sys.getrefcount(a))  # 2
```

引用计数增加的场景：
- 对象被创建：`a = 42`
- 被别名引用：`b = a`
- 被容器引用：`lst.append(a)`
- 作为函数参数传递：`func(a)`

引用计数减少的场景：
- 变量离开作用域
- 变量被重新赋值：`a = None`
- 从容器中移除：`lst.remove(a)`
- `del` 删除变量

**2. 标记清除（解决循环引用）：**

```python
a = []
b = []
a.append(b)
b.append(a)
del a, b
# 引用计数不为 0（互相引用），但已无法访问
# 标记清除可以回收这类对象
```

原理：从根对象（全局变量、栈、寄存器）出发遍历，标记所有可达对象，清除未标记对象。

**3. 分代回收（提高效率）：**

将对象分为三代：
- **0 代**：新创建的对象，回收频率最高（默认阈值 700）
- **1 代**：经历过一次 0 代回收的对象
- **2 代**：经历过一次 1 代回收的对象，回收频率最低

```python
import gc

# 查看阈值
print(gc.get_threshold())  # (700, 10, 10)

# 手动控制
gc.disable()   # 关闭自动回收
gc.enable()    # 开启自动回收
gc.collect()   # 手动触发回收
```

**4. 内存池（pymalloc）：**

小对象（< 512 字节）使用 pymalloc 分配，减少系统调用开销。大对象直接使用系统的 malloc。

**面试加分点：** Python 的内存管理不会立即归还给操作系统，而是保留在内存池中供后续使用。可以通过 `gc.collect()` 或 `malloc_trim()` 释放。

## 五、Python 的内存泄漏与排查

### 面试问题

Python 会发生内存泄漏吗？如何排查和解决？

### 标准答案

Python 虽然有垃圾回收，但仍然可能发生内存泄漏。

**常见原因：**

1. **循环引用 + `__del__` 方法**：有 `__del__` 的循环引用对象无法被分代回收
2. **全局变量/缓存无限增长**：
   ```python
   cache = {}
   def process(key, value):
       cache[key] = value  # 只增不减
   ```
3. **未关闭的资源**：文件、数据库连接、网络连接
4. **闭包持有大对象**：
   ```python
   def outer():
       big_data = [i for i in range(10**6)]
       def inner():
           return len(big_data)  # big_data 被闭包持有，无法释放
       return inner
   ```
5. **C 扩展中的内存泄漏**

**排查工具：**

```python
# 1. tracemalloc - 跟踪内存分配
import tracemalloc
tracemalloc.start()

# ... 你的代码 ...

snapshot = tracemalloc.take_snapshot()
for stat in snapshot.statistics('lineno')[:10]:
    print(stat)

# 2. objgraph - 查看对象引用关系
import objgraph
objgraph.show_most_common_types(limit=10)
objgraph.show_backrefs(objgraph.by_type('MyClass')[-1], max_depth=5)

# 3. gc 模块 - 查看不可回收对象
import gc
gc.set_debug(gc.DEBUG_LEAK)
gc.collect()
print(gc.garbage)  # 查看无法回收的对象
```

**解决策略：**

- 使用 `weakref` 弱引用打破循环引用
- 使用 `contextlib.closing()` 或 `with` 语句确保资源释放
- 缓存使用 `functools.lru_cache`（有大小限制）
- 定期清理全局字典/列表

## 六、Python 的 `__slots__` 与内存优化

### 面试问题

`__slots__` 是什么？为什么能节省内存？有什么限制？

### 标准答案

**问题背景：** 普通 Python 实例使用 `__dict__` 存储属性，这是一个字典，内存开销较大。

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

p = Point(1, 2)
print(p.__dict__)  # {'x': 1, 'y': 2}
# 每个实例都有一个完整的字典，占用大量内存
```

**使用 `__slots__`：**

```python
class Point:
    __slots__ = ('x', 'y')
    def __init__(self, x, y):
        self.x = x
        self.y = y

p = Point(1, 2)
# p.__dict__  # AttributeError: 没有 __dict__
# 属性直接存储在实例的固定内存空间中
```

**内存对比：**

```python
import sys

class Normal:
    def __init__(self, x, y):
        self.x = x
        self.y = y

class Slotted:
    __slots__ = ('x', 'y')
    def __init__(self, x, y):
        self.x = x
        self.y = y

n = Normal(1, 2)
s = Slotted(1, 2)
print(sys.getsizeof(n) + sys.getsizeof(n.__dict__))  # 约 200 字节
print(sys.getsizeof(s))                                # 约 64 字节
```

**限制：**

1. 使用 `__slots__` 的类不能动态添加属性：
   ```python
   s = Slotted(1, 2)
   s.z = 3  # AttributeError: 'Slotted' object has no attribute 'z'
   ```
2. 继承时子类需要重新定义 `__slots__`（否则子类仍会有 `__dict__`）
3. 不能与 `__weakref__` 一起使用（除非显式加入 slots）
4. 多继承时，多个父类都定义了 `__slots__` 会报错

**适用场景：** 创建大量实例（如百万级），且属性固定的场景。

## 七、Python 的可变对象与不可变对象

### 面试问题

Python 中可变对象和不可变对象有什么区别？函数参数传递时有什么影响？

### 标准答案

**核心区别：**

```python
# 不可变对象：修改会创建新对象
a = "hello"
b = a
a = "world"
print(b)  # "hello"  b 不受影响

# 可变对象：修改会直接影响原对象
a = [1, 2, 3]
b = a
a.append(4)
print(b)  # [1, 2, 3, 4]  b 也被修改了！
```

**函数参数传递：**

Python 的参数传递是**对象引用传递**（pass by object reference），不是值传递也不是引用传递。

```python
def modify_list(lst):
    lst.append(4)  # 修改了原对象

def modify_string(s):
    s = s + " world"  # 创建了新对象，原对象不变

my_list = [1, 2, 3]
modify_list(my_list)
print(my_list)  # [1, 2, 3, 4]  被修改了

my_str = "hello"
modify_string(my_str)
print(my_str)  # "hello"  没被修改
```

**常见的坑：**

```python
def foo(lst=[]):
    lst.append(1)
    return lst

print(foo())  # [1]
print(foo())  # [1, 1]  默认参数是可变对象，被共享了！
print(foo())  # [1, 1, 1]

# 正确写法
def foo(lst=None):
    if lst is None:
        lst = []
    lst.append(1)
    return lst
```

**面试加分点：** `int` 等小整数（-5 到 256）会被缓存，是同一个对象；大整数不会缓存。这是 CPython 的实现细节。

```python
a = 256
b = 256
print(a is b)  # True  缓存了

a = 257
b = 257
print(a is b)  # False（交互模式下）  没有缓存
```
