# MySQL优化方案

## 排查慢SQL的基本顺序

1. 开启并查看慢查询日志(查询看查询超过指定长时间的SQL)
2. 使用EXPLAIN + SQL 产看执行计划(是否使用索引，和查询的计划)
3. 优化索引，优化SQL语句
4. 连接池

## SQL的优化方向

- 索引
    - 是否使用索引？
    - 是否遵循最左原则？
    - 索引列是否使用函数？
    - 查询数据是否包含索引？(覆盖索引)
    - 索引是否失效？
- SQL语句
    - 减少查询没必要的字段 SELECT * 
    - 查询完后再排序
    - 分页查询
    - 减少JOIN的表数量
    - 多表查询的时候，使用小表驱动大表
    - 调整缓冲的大小
    - 最左原则
    - 减少函数再查询条件中使用
