# MySQL

- InnoDB的特性
    - 事务支持：ACID
        1. 原子性：事务中的多操作，要么全部执行成功，要么全部失败
        2. 一致性：单个事务执行前后的数据合法性和正确性。
        3. 隔离性：多个事务的并发运行存在数据访问问题
            * 读未提交：一个事务读取到另一个事务没有提交的数据（脏数据）
            * 读已提交：一个事务内部读读取多次数据，每次数据都不一样（不可重读取）
            * 不可重复读：一个事务中多次读取某个数据，只会显示第一次的数据(幻读)
            * 串行化：一个事务在修改的时候，其他事务排队依次执行
        4. 持久性：事务执行完成，提交之后会永久的保存下去，
    - 行锁：FROM UPDATE SKIP LOCKED
    -
  多版本并发控制（MVCC）：事务的每个写操作，不覆盖旧的数据，只是创建一个数据副本，读操作根据自己的隔离级别去读某个历史的数据，事务id，数据行id，旧版本数据的指针(
  undo log)
    - 聚集索引(B+树)：数据和索引存放在一起。
        1. 每一行都有一个数据片
        2. 每一个数据数据片都有一个子节点的id(下层节点最小值)
        3. 叶子节点存储完整的数据
        4. 所有的叶子节点是一个双向的链表，能够适用于和方便进行范围查找
    - 非聚集索引：索引树中不存数据，叶子节点只存，数据的主键，用过“回表”的方式，查询数据
    - 外键约束：确保子表要包含主表的，父表中已经存在的元素
- 慢查询如何分析和优化
    - 慢查询的分析：通过开启慢查询日志(SET GLOBAL slow_query_log = ON;)
    - 执行计划分析：通过关键之EXPLAIN
    - 优化索引：设计合适的索引内容，索引字段
    - 优化SQL语句的写法：
        - 索引优化：创建合适的索引
        - 索引覆盖，查询字段中最好包含索引
        - 避免使用： *，要查就是查所有对应的字段
        - 最左原则：索引字段放在最左边
        - 表连接：小表驱动大表，减少join量
        - 查询条件：尽量使用索引，减少函数在索引上使用，减少OR IN NOT IN 等函数使用
        - 分页查询：通过分页来减少数据量的查询
    - 优化表结构：设计合理的表结构，分库、分表，选择合适的数据引擎和数据字段
    - 使用连接池：避免频繁创建和释放连接，减少开销。

- MySQL的数据库的日志
    - Redo Log：写数据库的的时候先写Redo Log，保证数据的记录，保证事务的持久性，真正数据库已慢慢刷盘，MySQL崩溃的的时候可以通过Redo
      log来恢复一些数据。
    - Undo Log：用于多版本并发控制的时候，版本数据的回滚
    - Bin Log：记录的是所有数据的修改，用于数据的恢复和主从复制
    - Error Log：用于MySQL启动 关闭 等一些信息
    - Slow Query Log：慢查询日志，记录超过指定时间的查询记录
    - Relay Log：中继日志，用于用于主从复制中，从库读取数据，回放数据

- MySQL的主从复制的流程
    - 全量复制
    - 增量复制
        1. 主库将将写入的数据Binlog日志数据通过同步线程发送给从库
        2. 从库的 IO线程接收并读取Binlog日志，写成Relay Log(中间继日志)
        3. 从库的SQL线程 回放Relay Log中的SQL，并执行，实现数据插入和修改
