# Redis

- 常用的数据结构及其原理
    - String：SDS 动态字符串，记录容量和当前数据长度，以及字符数组
    - List：quicklist(双向链表+ ziplist/listpack)，多个压缩节点，每个节点存放多个元素
    - Hash：哈希表+压缩链表
    - Set：intset+哈希表，整数用intset 非整数用哈希表
    - Zset：字典和跳表，字典存数据 跳表用来score来排序


- 缓存的问题，击穿、穿透、雪崩
    - 穿透：访问一个不存在的失效或不存在的key
        - 缓存空
        - 布隆过滤器
    - 击穿：一个热点key突然过期，胆量请求打到数据库中
        - 互斥锁，只允许一个线程查询数据库，查看数据库迅速回填，其他吸纳成等待
        - 永不过期：设置一个很长的过期时间
    - 雪崩：同一时间当量key失效，大量请求直接打到数据库
        - 设置过期时间偏移量
        - 二级缓存
        - 限流
        - 预热缓存

- Redis的过期删除和淘汰策略
    - 过期删除
        1. 惰性删除：有请求来了，判断已经过期了，再删除
        2. 定时删除：过期了就立刻删除
        3. 定期删除：定期抽查一批key，进行删除
    - 淘汰策略
        1. 随机删除
        2. 最少访问
        3. 最长时间没被访问
        4. 最短过期时间
